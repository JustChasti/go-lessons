# Урок 3. Условия, циклы и рекурсия — практика

## Структура урока

```
l3/
├── README.md             — этот файл (описание задач здесь)
├── main.go               — точка входа, раскомментируй нужный блок
├── 01_conditions.go      — практика: условия    (main1)

└── 03_recursion.go       — практика: рекурсия   (main3)
```

Запуск из папки `l3`: `go run .`

В каждом файле задачи идут сверху вниз. У каждой — комментарий с именем функции, списком ожидаемых результатов и закомментированной заглушкой. Порядок работы стандартный:

1. Читаешь описание задачи в этом README.
2. В `.go`-файле раскомментируешь заглушку функции и реализуешь её.
3. В `main()` раскомментируешь соответствующие вызовы и запускаешь `go run .`.

## 1. Задачи по условиям

#### 1. `ticketPrice(age int, isStudent bool) int`

Цена билета в музей зависит от возраста и статуса студента:

- возраст 0–5: **бесплатно** (0)
- возраст 6–17: **детский** (300)
- студент 18+: **студенческий** (400)
- возраст 18–64: **взрослый** (800)
- возраст 65+: **пенсионный** (500)

Если возраст отрицательный — верни `-1` (признак некорректного входа). Студенческая скидка применяется только к взрослым: ребёнок 10 лет, даже формально «студент», всё равно платит детскую цену.

#### 2. `classifyTriangle(a, b, c int) string`

По трём сторонам верни одну из строк: `"невалидный"`, `"равносторонний"`, `"равнобедренный"`, `"разносторонний"`. Треугольник невалиден, если:

- любая сторона `<= 0`;
- сумма двух любых сторон не превышает третью (неравенство треугольника).

Сначала отсекаешь невалидный случай, потом классифицируешь оставшееся.

#### 3. `bmiCategory(weightKg, heightM float64) string`

Индекс массы тела считается как вес в килограммах, делённый на рост в метрах в квадрате. Верни категорию:

- `< 18.5` → `"недобор"`
- `18.5` включительно до `25` → `"норма"`
- `25` включительно до `30` → `"лишний вес"`
- `>= 30` → `"ожирение"`

Если вес или рост `<= 0` — верни `"некорректно"`.

#### 4. `nextLight(current string) string`

По текущему сигналу светофора верни следующий: `"red"` → `"green"`, `"green"` → `"yellow"`, `"yellow"` → `"red"`. Для любого другого входа — `"error"`. Задача идеально ложится на `switch`.

#### 5. `dayKind(day int) string`

Классификация дня недели по номеру: 1 — понедельник, …, 7 — воскресенье. Для 1–5 верни `"рабочий"`, для 6 и 7 — `"выходной"`. Всё остальное — `"некорректно"`.

## 2. Задачи по циклам

#### 1. `sumOfMultiples(limit, k int) int`

Сумма всех чисел от 1 до `limit` включительно, которые делятся на `k`. Если `k <= 0` — верни 0.

#### 2. `countDigits(n int) int`

Сколько цифр в записи целого числа. Для отрицательных минус не считается. Число разрядов заранее неизвестно — отсюда цикл.

#### 3. `collatzSteps(n int) int`

Последовательность Коллатца: для чётного `n` берём `n/2`, для нечётного — `3n+1`. Посчитай, за сколько шагов от стартового `n` доберёшься до 1. Для единицы — 0 шагов. Для `n <= 0` верни `-1`.

#### 4. `isPalindromeStr(s string) bool`

Строка — палиндром, если читается одинаково в обе стороны. Регистр игнорируй. Пустая строка и строка из одного символа считаются палиндромом. Подсказка: приведи строку к `[]rune` и иди двумя указателями с концов к середине. Для приведения к нижнему регистру используй пакет `strings` или преобразование рун через `unicode`.

#### 5. `minMax(nums []int) (int, int)`

Минимум и максимум среза за **один** проход. Для пустого среза верни `(0, 0)`. Два раза по срезу не ходить — это важный принцип.

#### 6. `smallestPowerGreater(limit int) int`

Наименьшее `k`, при котором `2^k > limit`. Если `limit < 1` — верни 0. Формулу через логарифм не использовать

## 3. Задачи по рекурсии

#### 1. `power(base, exp int) int`

Возведение в неотрицательную целую степень. Определение прямо ложится в код:

- `base^0 = 1`
- `base^n = base * base^(n-1)`

#### 2. `gcd(a, b int) int`

Наибольший общий делитель по алгоритму Евклида:

- `gcd(a, 0) = a`
- `gcd(a, b) = gcd(b, a mod b)`

Гарантируется, что `a >= 0` и `b >= 0`.

#### 3. `sumDigitsRec(n int) int`

Сумма цифр числа через рекурсию. Отрицательные считаем как положительные:

- если `n < 10`, то ответ — сам `n`
- иначе — `(n mod 10) + sumDigitsRec(n / 10)`

#### 4. `reverseRec(s string) string`

Рекурсивное обращение строки:

- пустая строка остаётся пустой
- иначе — `reverseRec(s[1:])` + первый символ

Чтобы корректно работать со строками произвольного содержимого, приведи к `[]rune` — иначе на юникоде поедешь по байтам.

#### 5. `countPaths(rows, cols int) int`

Робот стартует в левом верхнем углу сетки `rows × cols` и должен попасть в правый нижний. Ходить можно только вправо или вниз. Сколько всего разных путей?

- если `rows == 1` или `cols == 1` — путь единственный (1)
- иначе — `countPaths(rows-1, cols) + countPaths(rows, cols-1)`

Когда напишешь — попробуй вызвать `countPaths(20, 20)` и посмотри, сколько оно будет думать. Это хорошая иллюстрация, почему наивная рекурсия без запоминания результатов — не всегда то, что нужно на практике.
