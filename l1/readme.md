## Домашка

В первом уроке мы ещё не касаемся модулей и многофайловых проектов — поэтому всё пишем в одном файле. Открой `main.go`, и тело каждого задания клади **внутрь функции `main`** — той самой, с которой стартует программа. Запуск стандартный: `go run main.go`.

Если хочешь сразу решить все четыре задания — просто пиши их подряд в `main`, разделяя комментариями. Если хочется проверять по одному — закомментируй те, что пока не делаешь.

#### 1. Визитка

Объяви три переменные через короткое присваивание `:=`: `name` со своим именем, `age` с возрастом, `city` с городом. Тип Go выведет сам — это и есть смысл `:=`. Выведи их одной строкой через `fmt.Printf` в формате `"Меня зовут <имя>, мне <возраст> лет, я из <город>"`. Под каждый тип в `Printf` свой глагол: `%s` для строки, `%d` для целого числа. Не забудь `\n` в конце, чтобы следующий вывод начался с новой строки.

#### 2. Конвертер температуры

Дано: переменная `celsius` со значением `36.6`. Нужно посчитать температуру в фаренгейтах по формуле `F = C * 9/5 + 32` и вывести результат.

Результат объяви как `float64` — то есть так же, как `celsius`. Если в `9/5` оба числа целые — Go посчитает их целочисленно и получится `1`, что сломает формулу. Поэтому либо пиши `9.0/5.0`, либо считай через `* 1.8`. Это первая ловушка типов: дробное деление в Go получается только когда хотя бы один из операндов дробный.

Вывод — через `fmt.Printf` с двумя знаками после запятой: `%.2f`. Ожидаемая строка для `36.6` — `36.60°C = 97.88°F`.

#### 3. Обмен переменных

Даны `x := 10` и `y := 20`. Нужно поменять их значения местами — так, чтобы после обмена `x` стало `20`, а `y` стало `10`. И главное условие: **без третьей переменной**.

В большинстве языков для этого используют временную переменную: `tmp = x; x = y; y = tmp`. В Go так тоже можно, но язык даёт более короткий способ — множественное присваивание: `x, y = y, x`. Справа от `=` сначала вычисляются оба значения целиком, и только потом они присваиваются левым переменным. Поэтому никакой третьей переменной не требуется — её роль играет сама механика языка.

Перед обменом выведи `До обмена: x=10, y=20`, после — `После обмена: x=20, y=10`. Глагол для целых чисел в `Printf` — `%d`.

#### 4. Определи тип

Объяви через `:=` пять переменных:

- `a := 42`
- `b := 3.14`
- `c := "hello"`
- `d := true`
- `e := 'A'`

Выведи тип каждой через `fmt.Printf` с глаголом `%T` — он печатает не значение, а тип переменной. Например, `fmt.Printf("a: %T\n", a)` напечатает `a: int`.

Главный сюрприз — переменная `e`. На вид это просто символ, но в Go её тип будет `int32`, и под этим именем скрывается отдельный псевдоним — `rune`. Одинарные кавычки в Go — это не «тип char», как в некоторых языках, а **рунный литерал**: то есть код символа Unicode, целое число. `'A'` — это `65`. Это важно знать заранее: когда мы дойдём до строк и юникода, понимание, что одинарные кавычки = число-код, а двойные = строка байтов, сэкономит много замешательства.
