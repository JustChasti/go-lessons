## 1. Условия

### if / else

Проверка условия — самый базовый способ управлять ходом программы. Две особенности синтаксиса:

- Скобки вокруг условия **не пишутся**: просто `if` и выражение.
- Фигурные скобки **обязательны всегда**, даже если внутри одна строка.

Открывающая `{` должна стоять на той же строке, что и `if` — это правило компилятора, а не стиль.

### Короткая форма с инициализацией

Характерная для Go конструкция: переменную можно объявить прямо внутри `if`, и она будет жить только в пределах `if/else` блока. Чаще всего так проверяют результат функции, который дальше уже не нужен.

### switch

Удобный способ сравнить значение с несколькими вариантами. Три ключевых момента:

- После сработавшего `case` Go **автоматически** выходит из `switch` — `break` писать не надо.
- В одном `case` можно перечислить несколько значений через запятую.
- Можно написать `switch` **без выражения** — тогда каждый `case` содержит булево условие. Это прямая замена длинной цепочки `if / else if`.

## 2. Циклы

В Go единственный цикл — это `for`. У него три формы, которые закрывают все сценарии.

**Классическая** — с инициализацией, условием и шагом. Подходит, когда нужен счётчик.

**Только условие** — используется, когда количество итераций заранее неизвестно, а выход определяется каким-то условием.

**Без условия** — бесконечный цикл, выход только через `break` изнутри. Нужен для серверов, опросов, REPL-ов — всего, где выход определяется логикой, а не счётчиком.

### break и continue

- `break` — выйти из цикла.
- `continue` — перейти к следующей итерации, пропустив остаток текущей.

### for range

Для обхода коллекций. Возвращает пары `(индекс, значение)` для срезов, массивов и строк; пары `(ключ, значение)` для мап. Если одна из переменных не нужна — используй `_`.

По строке `range` идёт **не по байтам, а по рунам** — в UTF-8 один символ может занимать несколько байт, и `range` это учитывает. Индекс при этом всё равно байтовый. Важный момент, к которому ещё вернёмся.

## 3. Функции

Функция — именованный блок кода с параметрами и результатами.

### Объявление

Параметры описываются как `имя тип`. Если у нескольких соседних параметров тип совпадает, его можно указать один раз: вместо `(a int, b int) int` пишут `(a, b int) int`.

### Несколько возвращаемых значений

Одна из самых характерных особенностей Go: функция может вернуть несколько значений сразу. На практике почти всегда возвращают пару `(результат, ошибка)`. Если ошибки нет — возвращают `nil` вторым значением. Это идиома, которая пронизывает весь Go — видно в любом коде.

### Именованные возвращаемые значения

В сигнатуре можно задать имена возвращаемым переменным. Тогда они автоматически объявлены внутри функции, а `return` без аргументов возвращает их текущие значения. Удобно в коротких функциях; в длинных «голый» `return` только путает.

### Вариативные функции

Параметр с `...` принимает любое количество аргументов одного типа. Внутри функции такой параметр — обычный срез. При вызове можно передать сколько угодно значений через запятую, или распаковать готовый срез оператором `...` после имени.

### Функции — это значения

Функцию можно присвоить в переменную, передать в другую функцию аргументом, вернуть из функции результатом. Тип такой переменной описывается её сигнатурой: например, `func(int, int) int` — «функция, принимающая два `int` и возвращающая `int`».

### Анонимные функции и замыкания

Функцию можно объявить прямо на месте, без имени. Если такая функция использует переменные из окружающего контекста — она становится **замыканием**: захваченные переменные продолжают жить вместе с функцией, даже после того, как внешняя функция завершилась.

### Рекурсия

Функция может вызывать сама себя. У любой рекурсии должна быть **база** — условие, при котором функция возвращает результат напрямую, не вызывая себя снова. Без базы — бесконечный вызов и переполнение стека.
